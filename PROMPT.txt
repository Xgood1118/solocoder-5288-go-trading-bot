需求：Go 加密货币交易机器人（教学版），多策略 / 订单簿 / 回测 / 实时 / 风控 / 并行。

背景：组里研究量化交易时做了简化版加密货币交易机器人（不接真交易所，mock 一个交易所）。重点是：策略引擎、回测系统、订单簿管理、实时行情、风控、并行执行。技术栈 Go + chi + SQLite + WebSocket（mock 行情源）。

核心功能：
- 多策略：均线交叉 / 海龟交易 / RSI / MACD / Bollinger Bands / 网格交易，每种独立配置
- 订单簿：买卖盘、价格深度、撮合（mock）
- 行情：实时 WebSocket 推送（mock 推送器，生成正弦波 + 噪声 mock 行情）
- 回测：用历史数据（mock 数据集）跑策略，输出夏普 / 最大回撤 / 胜率
- 实时：选策略 → 订阅行情 → 触发信号 → 下单 → 监控
- 下单：限价单 / 市价单 / 止损单 / 止盈单
- 风控：单笔限额、累计亏损熔断、最大持仓、禁用币种
- 并行：多策略并行跑，每个策略独立 goroutine
- 日志：策略日志、订单日志、错误日志
- 报表：每日盈亏、持仓、成交明细

涉及模块：
- `internal/strategy/`：策略接口、各类策略实现
- `internal/market/`：行情、订单簿、撮合
- `internal/backtest/`：回测引擎
- `internal/live/`：实时交易
- `internal/order/`：订单管理、状态机
- `internal/risk/`：风控规则
- `internal/position/`：持仓、盈亏计算
- `internal/report/`：报表
- `cmd/bot/main.go`：CLI 入口
- `web/`：监控 dashboard（实时持仓 / 盈亏 / 策略状态）

注意事项：
- 策略接口：`type Strategy interface { OnTick(price) Signal; OnInit(); OnStop() }`
- 订单簿用红黑树（标准库 `container/list` 或自己实现）维护买卖盘
- 撮合：限价单进订单簿，市价单立即按当前最优价成交
- 回测要避免"未来函数"：用历史 T-1 时刻的数据做决策，结果用 T 时刻价
- 回测要考虑滑点（默认 0.1%）、手续费（默认 0.1%），不要假设完美成交
- 多策略并行：每个策略订阅同一行情源，行情推到来时按顺序执行（不要 race）
- 共享数据（账户余额、持仓）要加锁（sync.RWMutex）
- 风控在订单前校验：单笔限额、累计亏损、持仓上限
- 熔断：当日累计亏损 > 5% 停止当日所有交易
- 实时行情用 goroutine 订阅，disconnect 重连 + 错过的行情补单（mock）
- 持久化：策略配置 / 订单 / 持仓 / 报表存 SQLite
- 时间同步：交易所时间戳 vs 本地时间，行情处理用交易所时间排序
- 金额用 int64（最小单位 satoshi），不要用 float
- 钱包地址（mock）：每个用户一个 mock 地址，余额变动写流水
- 风险敞口计算：Σ(持仓 × 实时价)，超过阈值报警
- 日志：结构化（slog），包含策略 ID / 时间戳 / 价格 / 信号 / 动作
- 测试：策略 / 风控 / 撮合 用 mockall + 标准 testing
- 监控：Prometheus 指标（订单数 / 成交额 / 盈亏）
- 部署：单二进制 + systemd 服务，dashboard 走独立端口
- 安全：API key 加密存（aes-gcm），从环境变量读
- 异常处理：网络断 / 行情乱序 / 撮合失败都要 graceful 恢复
